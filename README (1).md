<h2><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction</h2>
<p>This is a Theano/Blocks implementation of the Attention Sum Reader model as presented in "Text Comprehension with the Attention Sum Reader Network" available at <a href="http://arxiv.org/abs/1603.01547" rel="nofollow">http://arxiv.org/abs/1603.01547</a>.
We encourage you to familiarize yourself with the model by reading the above article prior to studying the particulars of this implementation.</p>
<h1><a id="user-content-quick-start" class="anchor" aria-hidden="true" href="#quick-start"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Quick start</h1>
<p>If you want to get started as fast as possible try this:</p>
<pre><code>./prerequisites.sh
cd asreader
./quick-start-cbt-ne.sh
</code></pre>
<p>If you do not have a GPU available, remove the device=gpu flag from quick-start-generic.sh. However note that training the text comprehension tasks on a CPU is likely to take a prohibitively long time.</p>
<p>This should install the prerequisites, download the CBT dataset, train two models on the named-entity part of the data, form an ensemble and report the accuracies.</p>

<p>Provided you have python with pip installed, running</p>
<pre><code>prerequisites.sh
</code></pre>
<p>should install Blocks and dependencies for you.
It also downloads the Children's Book Test dataset and the CNN and Daily Mail news datasets. We are aware that
the news data download sometimes crashes. Rerunning the script prepare-rec-data.sh should be able to resume

<h2><a id="user-content-getting-data" class="anchor" aria-hidden="true" href="#getting-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Getting data</h2>
<h3><a id="user-content-children-book-test" class="anchor" aria-hidden="true" href="#children-book-test"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Children Book Test</h3>
<p>Children Book Test data should be already downloaded by the quick start script. If you skipped this script you can prepare the data by <code>prepare-cbt-data.sh</code></p>
<h3><a id="user-content-cnn-and-daily-mail" class="anchor" aria-hidden="true" href="#cnn-and-daily-mail"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>CNN and Daily Mail</h3>
<p>The best way how to get the CNN and DailyMail datasets is to download the questions and stories files from <a href="http://cs.nyu.edu/~kcho/DMQA/" rel="nofollow">http://cs.nyu.edu/~kcho/DMQA/</a>.
Place them into folder <code>$CLONE_DIR/data</code> and run a script <code>$CLONE_DIR/data/prepare-rc-data-downloaded.sh</code>.</p>
<p>Alternatively you can use a script <code>$CLONE_DIR/data/prepare-rc-data.sh</code> that downloads the data using the original scripts from <a href="https://github.com/deepmind/rc-data">https://github.com/deepmind/rc-data</a>. However,
the news data download sometimes crashes. Therefore it is often necessary to download missing articles by re-running <code>generate_questions.py</code> script.</p>
<p>Now when you have CNN and DailyMail datasets you can use them to train the models:</p>
<pre><code>cd asreader
./quick-start-cnn.sh
./quick-start-dm.sh
</code></pre>
<h2><a id="user-content-training-models" class="anchor" aria-hidden="true" href="#training-models"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Training models</h2>
<p>The model can be trained by running the <code>text-comprehension/as_reader.py</code> script. The simplest usage is:</p>
<pre><code>python text_comprehension/as_reader.py --dataset_root data/CBTest/data/ --train train.txt --valid valid.txt --test test.txt
</code></pre>
<p>where the .txt files are the appropriate datasets. Some of the recommended configurations can be copied from the quick-start-cbt-ne.sh script</p>
<p>You may need to prepend the following prefixes in front of the command and run it from the project root directory</p>
<pre><code>THEANO_FLAGS="floatX=float32,device=gpu" PYTHONPATH="$PYTHONPATH:./"
</code></pre>
<p>Some of the most useful command line arguments you may wish to use are the following</p>
<ul>
<li>--dataset_type [cbt|cnn|babi] - the type of dataset that is being used. Defaults to the Children's Book Test</li>
<li>-b 32 ... batch size - larger values usually speed up training however increase the memory usage</li>
<li>-sed 256 ... source embedding dimension</li>
<li>-ehd 256 ... the number of hidden units in each half of the bidirectional GRU encoders</li>
<li>-lr 0.001 ... learning rate</li>
<li>--output_dir ... output directory for the validation and test prediction files</li>
<li>--patience_metric accuracy ... when this metric stops improving, training is eventually stopped</li>
<li>-p 1 ... the number of epochs for which training continues since achieving the best value of the patience metric</li>
<li>--own_eval ... runs a script that eb</li>
<li>--append_metaparams ... includes the metaparameters in the filename of the generated prediction files - useful when generating multiple models</li>
<li>--weighted_att ... instead of attention sum, use the weighted attention model to which we compare the ASReader in the paper
The full list of parameters with descriptions can be displayed by running the script with the -h flag.</li>
</ul>
<h2><a id="user-content-ensembling" class="anchor" aria-hidden="true" href="#ensembling"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Ensembling</h2>
<p><code>as_reader.py</code> can generate the predictions for the test and validation datasets into the output directory. By default the predictions are generated every epoch. The text_comprehension/eval/copyBestPredictions directory can then be used to find the time at which model achieved the best validation accuracy and it copies the corresponding validation and test predictions to a separate folder.
An example syntax is</p>
<pre><code>python text_comprehension/eval/copyBestPredictions.py -vp cbtest_NE_valid_2000ex.txt. -tp cbtest_NE_test_2500ex.txt. -i out_dir -o out_dir/best_predictions
</code></pre>
<p>where <code>-vp</code> and <code>-tp</code> give the prefixes of the validation and test predictions respectively. These are usually the validation and test dataset filenames.</p>
<p>Once the <code>best_predictions</code> directory contains only one test and one validation prediction for each model, we can fuse these using the <code>text_comprehension/eval/fusion.py</code> for instance using the following command:</p>
<pre><code>python text_comprehension/eval/fusion.py -pr "out_dir/best_predictions/*.y_hat_valid" -o $OUT_DIR/best_predictions/simple_fusion.y_hat -t foo --fusion_method AverageAll
</code></pre>
<p>where <code>-pr</code> gives an expression for the validation predictions to be used and <code>-o</code> specifies the file to output.</p>
<p>The script provides three methods of fusion toggled by the <code>--fusion_method</code> parameter:</p>
<ul>
<li><code>AverageAll</code> - the ensemble prediction is a mean of all the supplied single-model predictions</li>
<li><code>pBest</code> - sorts the candidate models by validation accuracy and selects the best proportion p of models to form the ensemble</li>
<li><code>AddImprover</code> - sorts the candidate models by validation accuracy and then tries adding them to the ensemble in that order
keeping each model in the ensemble only if it improves its val. accuracy</li>
</ul>
